# 个人博客简易更新指南 (本地模式)

既然线上登录遇到困难，我们推荐使用**最稳定、最简单**的**本地可视化编辑模式**。
这种方式**不需要**复杂的 GitHub 授权，**不需要**配置密码，且**完全免费**。

## 核心流程
1. **本地写作**：在你的电脑上打开可视化后台写文章。
2. **自动保存**：Keystatic 会自动把文章保存为项目文件。
3. **一键发布**：通过 VS Code 将变动推送到 GitHub，网站自动上线。

---

## 详细步骤

### 第一步：启动后台
在 VS Code 的终端中输入以下命令启动项目（如果你还没启动）：
```powershell
npm run dev
```

### 第二步：打开编辑器
在浏览器访问：
👉 **http://localhost:4321/keystatic**

*   注意是 `localhost`，不是线上的网址。
*   **不需要登录**，直接就能用。

### 第三步：撰写内容
1.  点击 **Blog** 或 **Projects**。
2.  新建或编辑文章。
3.  上传图片、排版文字。
4.  点击右上角的 **Save** (保存)。
    *   *此时，你的文章已经自动保存到了电脑的 `src/content` 文件夹中。*

### 第四步：发布上线
写好后，回到 VS Code：

1.  点击左侧侧边栏的 **源代码管理 (Source Control)** 图标（看起来像树枝的分叉）。
2.  在输入框输入备注，例如：“新增一篇博客”。
3.  点击 **提交 (Commit)** 按钮。
4.  点击 **同步更改 (Sync Changes)** 按钮。

**完成！** 🚀
GitHub 收到你的推送后，Vercel 会自动开始构建，几分钟后你的线上网站就会更新。

---

## 这种方式的好处
*   ✅ **永不报错**：避开了复杂的网络验证和权限问题。
*   ✅ **数据安全**：所有文章数据都在你本地电脑上，不会丢失。
*   ✅ **体验一致**：拥有和线上一样的可视化编辑体验。
